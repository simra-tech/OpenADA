# First-session task

## Scenario

A self-described vibe designer starts with a fresh workspace and no assumed
ASIC setup. Codex must onboard and execute the work without asking the user to
run commands. The task deliberately tests autonomous target selection,
provisioning, implementation, evidence handling, and honest stopping behavior.

## Sanitized task text

> Use the installed `$openada:openada` skill. I am a vibe designer and want you
> to take this fresh workspace from nothing to a minimal CPU tapeout candidate
> using an appropriate open-source PDK, preferably IHP SG13G2. Onboard the
> project, determine and provision the exact tools and PDK you need, and keep
> going without asking me to execute anything. Create a small but real CPU,
> verify its RTL, synthesize and physically implement it, include a padframe
> only if the available open collateral supports one honestly, run timing, DRC,
> and LVS, and preserve auditable evidence. Treat PDKs and shared sources as
> read-only. Do not call a preview driver foundry signoff and do not claim
> tapeout readiness for any unverified stage. Continue until the candidate is
> complete or you have exhausted safe recovery paths; if OpenADA lacks a
> semantic operation, record that gap and use a clearly labeled native tool
> only when necessary.

No credentials, authentication material, private repository content, or user
identity is included in this record.
